# Knowledge: Frontend ROI & User Value

Every `IMPLEMENTATION_PLAN.md` must contain a 'Business Value' section.

## 1. ROI Formula
- **ROI (%) = [(Total Benefits - Total Costs) / Total Costs] * 100**
- **Payback Period:** Time required to recover the initial investment.

## 2. Variables
- **Costs:** Engineering hours, asset management, complexity introduction (e.g., state management).
- **Benefits:** Lighthouse score improvement (Performance, A11y), user conversion rate, bounce rate reduction.

## 3. Frontend Value (Experience & Conversion)
Beyond direct ROI, the Frontend Architect must evaluate:
- **Conversion Impact:** Does this change reduce friction in the user journey?
- **Retention (Stickiness):** Does the improved performance/UI keep users on the app longer?
- **SEO Value:** Better Web Vitals (LCP, CLS, INP) lead to higher search ranking.
- **A11y Compliance:** Avoiding legal risks and expanding the user base (e.g., WCAG compliance).

## 4. Performance as a Feature
- **Loading Speed:** Every 100ms improvement can lead to a 1% increase in conversion.
- **Perceived Performance:** Skeleton loaders, optimistic UI updates.
- **Interactivity:** Reduction in TBT (Total Blocking Time) increases user satisfaction.

## 5. Risk Buffer
- **20% Buffer:** Always add a 20% margin to cost estimates for "unknown-unknowns."
- **Risk Assessment:** Categorize changes as Low, Medium, or High risk based on cross-browser and device fragmentation.
