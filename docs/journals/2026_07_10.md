- # Upgraded Squad Full-Sync Protocol & Agent squad-docs Commands
	- Worked on upgrading the project-wide documentation synchronization workflow (`/squad:full-sync`) to produce PRD-level detailed, template-compliant pages.
	- Created three new Logseq templates under `common/templates/logseq/` to define the high-fidelity depth standard for each documented feature type:
		- [[business_flow_page.md]] - Template for business flows, actor maps, Journeys, and Gherkin ACs.
		- [[technical_endpoint.md]] - Template for API endpoints, schemas, validation, execution flow, data access, and resilience.
		- [[technical_data_model.md]] - Template for database models, entity properties, relationships, and performance indexes.
	- Created a new shared skill file [[common/skills/squad_docs.md]] containing the **Squad Documentation Protocol**. It mandates a manifest-first scanning phase, per-entity detailed file generation, template compliance, and AST code citations for all documentation tasks.
	- Created new isolated `squad-docs` commands for all 8 participating agents, ensuring that development-time documentation workflows remain completely unaffected:
		- `po/commands/po/squad-docs.toml`
		- `researcher/commands/researcher/squad-docs.toml`
		- `architect/commands/architect/squad-docs.toml`
		- `backend/commands/backend/squad-docs.toml`
		- `frontend/commands/frontend/squad-docs.toml`
		- `mobile/commands/mobile/squad-docs.toml`
		- `forge/commands/forge/squad-docs.toml`
		- `decoder/commands/decoder/squad-docs.toml`
	- Updated [[squad/commands/squad/full-sync.toml]] to invoke these new `/agent:squad-docs` commands instead of `/agent:docs`.
	- Registered the new features and nodes in the [[Global Symbol Registry]] ([[docs/pages/registry.md]]) and updated [[TECHNICAL_SPECS]] and [[code-dna]].
